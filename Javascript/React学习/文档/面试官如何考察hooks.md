面试官问 Hooks，绝不是想听你背诵 API。他们想考察你对 React 工作方式的理解深度、你的编程思想以及你解决实际问题的能力。

问题可能分为以下几个层次：

#### 层次一：基础知识 (检查你是否用过)

- "请介绍一下 React Hooks，它解决了什么问题？"
    - **回答要点**：对比 Class 组件的痛点（`this`、逻辑复用、复杂性），引出 Hooks 的优势（函数式、逻辑聚合、易于复用）。
- "请介绍一下 `useState` 和 `useEffect` 的用法。"
    - **回答要点**：`useState` 用于状态管理，返回 `[value, setValue]`。`useEffect` 用于处理副作用，介绍它的依赖数组（`[]`, `[deps]`, 无）分别代表什么场景（仅挂载时、依赖变化时、每次渲染时）以及如何返回一个清理函数。
- "React Hooks 的使用规则是什么？为什么有这些规则？"
    - **回答要点**：说出两条规则。**重点解释“为什么”**，这能体现你的理解深度。你需要提到我们上面讨论的“**依赖调用顺序的数组模型**”。能画图或清晰描述这个模型会非常加分。

#### 层次二：进阶理解 (检查你是否深入思考过)

- "`useCallback` 和 `useMemo` 有什么区别？什么时候应该使用它们？"
    - **回答要点**：`useMemo` 缓存计算结果（值），`useCallback` 缓存函数本身（引用）。它们都用于性能优化，防止不必要的子组件重渲染。要强调**不要滥用**，因为它们本身也有开销。
- "`useRef` 和 `useState` 有什么不同？"
    - **回答要点**：`useState` 的值改变会触发重渲染。`useRef` 返回一个可变的 `ref` 对象，其 `.current` 属性的改变**不会**触发重渲染。它主要用于访问 DOM 元素，或存储一个在多次渲染间需要保持不变、但又不想触发渲染的值（如定时器 ID）。
- "请解释一下 `useEffect` 中的闭包陷阱（Stale Closure）。"
    - **回答要点**：如果在 `useEffect` 中使用了某个 state 或 prop，但没有把它加入依赖数组，那么 effect 内部的函数就会“捕获”到第一次渲染时的旧值。这是一个非常经典的进阶问题，能回答好说明你经验丰富。

#### 层次三：实践与架构 (检查你解决问题的能力)

- "如果让你实现一个获取用户数据的 Hook，你会怎么做？（`useFetch`）"
    - **回答要点**：这是一个开放题。你需要设计一个自定义 Hook `useFetch(url)`，它内部使用 `useState` 来管理 `data`, `loading`, `error` 三个状态，并使用 `useEffect` 来执行 `fetch` 请求。要考虑取消请求的清理逻辑。
- "你如何在一个大型应用中管理全局状态？"
    - **回答要点**：这会引出 `useContext` + `useReducer` 的组合，或者更成熟的状态管理库如 Redux, Zustand, MobX。这表明你不仅了解组件内部状态，还思考过组件间的通信和整个应用的状态架构。
- "你有没有封装过自己的自定义 Hook？可以举个例子吗？"
    - **最佳回答**：准备一个你封装过的、有实际意义的自定义 Hook，比如 `useLocalStorage`（同步 state 到 localStorage）、`useWindowSize`（监听窗口尺寸变化）或 `useDebounce`（处理防抖）。这能极大地展示你的代码抽象和复用能力。

**面试建议：** 在准备面试时，不仅要记住 API，更要亲手用 Hooks 写一些小项目，特别是封装几个实用的自定义 Hook。在回答问题时，尽量结合你做过的项目或遇到的实际问题来举例，这样会让你的回答更生动、更有说服力
