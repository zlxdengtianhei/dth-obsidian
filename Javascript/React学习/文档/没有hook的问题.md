- **`this` 关键字的困扰**：在 JavaScript 的 Class 中，`this` 的指向非常灵活，开发者需要手动绑定事件处理函数（如 `this.handleClick = this.handleClick.bind(this)`），这既繁琐又容易出错。
- **逻辑复用困难（“包装器地狱”）**：为了在多个组件间复用相同的逻辑（比如，订阅数据源），社区发明了“高阶组件”（HOC）和“渲染属性”（Render Props）等模式。这些模式虽然能用，但会导致组件树层层嵌套，形成所谓的“包装器地狱”（Wrapper Hell），让代码难以阅读和理解。
- **复杂组件难以维护**：在 Class 组件中，相关的逻辑往往被迫分散在不同的生命周期方法里。例如，一个数据获取的逻辑，可能需要在 `componentDidMount` 中订阅，在 `componentWillUnmount` 中取消订阅。而一个完全不相关的逻辑（比如设置一个定时器）也可能混在这两个方法里，导致代码混乱。
- **Class 对人和机器都不友好**：Class 的概念对初学者有一定门槛，同时也不利于代码的压缩和优化。

**Hooks 的目标就是解决以上所有问题**：让你用更简洁的函数，在函数式组件中实现所有 Class 组件能做到的事情，并且能更好地组织和复用逻辑。
